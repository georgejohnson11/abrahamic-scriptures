# Abrahamic Scriptures

![License](https://img.shields.io/badge/license-MIT-blue) ![Node.js](https://img.shields.io/badge/node-%3E%3D16.0.0-green) ![Status](https://img.shields.io/badge/status-active-success)

A full-stack web application for exploring Abrahamic scriptures — the Bible and the Quran. Provides a modern, responsive interface with comprehensive API endpoints for scripture search and retrieval across multiple translations and commentaries.

## Quick Start

### Local Development

**Prerequisites**: Node.js 16+, npm

```bash
# Install dependencies and build frontend
npm install
npm run dev              # Run both server and client in dev mode

# OR for production
npm start               # Build client and run server on http://localhost:3001
```

Visit **http://localhost:3001** to access the application (or **http://localhost:5173** in dev mode for the client).

---

## Architecture

- **Frontend**: React + Vite, built to `/dist` folder
- **Backend**: Node.js + Express API, served from root on port 3001
- **Static Assets**: Served by Express from `/dist` directory
- **Databases**: SQLite
  - `db/bible.db` — King James Bible with book/chapter/verse structure
  - `db/quran.db` — Quran with Arabic text, surah metadata, and tafseer (commentary)

## API Endpoints

### Bible
- `GET /api/bible/books` — list all books
- `GET /api/bible/books/:id` — book info
- `GET /api/bible/books/:id/chapters-count` — chapter count
- `GET /api/bible/books/:id/chapters/:ch/verses` — verses
- `GET /api/bible/search?q=` — search verses

### Quran
- `GET /api/quran/surahs` — list all surahs
- `GET /api/quran/surahs/:id/verses` — verses for surah
- `GET /api/quran/tafseer-books` — available tafseer books
- `GET /api/quran/tafseer/:surah/:book/:verse` — tafseer text
- `GET /api/quran/search?q=` — search Quran text

## Key Design Decisions

- **Unified Deployment**: Backend serves both API and static frontend from a single port
- **Build Process**: `npm install` installs all dependencies, `npm start` builds the client then runs the server
- **Static Serving**: Express serves built files from `/dist` directory
- **Database Location**: SQLite files stored in `/db` directory at the root level

## Deployment

### Docker

#### Build Image

```bash
docker build -t abrahamic-app:1.0 .
```

#### Run Container

```bash
docker run -p 3001:3001 abrahamic-app:1.0
```

The application will be available at `http://localhost:3001`

### Build for Production

```bash
npm install
npm start
```

This will build the frontend to `/dist` and start the server on port 3001.

## Development

### Available Scripts

- `npm start` — Build frontend and run server on port 3001
- `npm run dev` — Run server and dev client in parallel with hot reload
- `npm run build:client` — Build frontend only
- `npm run dev:server` — Run server with auto-restart on file changes
- `npm run dev:client` — Run client dev server with hot reload

### Adding New Features

1. **Backend**: Extend API routes in `server.js`
2. **Frontend**: Add React components in `client/src/pages/` or `client/src/`
3. **Database**: Modify query logic in `db/bible.js` or `db/quran.js` (in root)

### Testing Locally

```bash
# Test backend API directly
curl http://localhost:3001/api/bible/books

# Test in browser
open http://localhost:3001
```
